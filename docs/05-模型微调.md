# 模型微调教程

本教程介绍如何使用 MLX-LM 对大语言模型进行 LoRA 或 QLoRA 微调。

## 支持的模型家族

- Mistral / Mixtral
- Llama 系列 (Llama 2, Llama 3, Llama 3.1, Llama 3.2)
- Qwen / Qwen2 / Qwen2.5
- Phi2 / Phi3
- Gemma / Gemma 2
- OLMo
- MiniCPM
- InternLM2

## 前置准备

### 安装训练依赖

```bash
pip install "mlx-lm[train]"
```

这将安装：
- `datasets` - Hugging Face 数据集
- `tqdm` - 进度条显示

## 数据准备

### 数据目录结构

```
data/
├── train.jsonl   # 训练数据（必需）
├── valid.jsonl   # 验证数据（可选）
└── test.jsonl    # 测试数据（评估时必需）
```

### 数据格式

#### 1. 聊天格式 (chat)

最常用的格式，适合对话类数据：

```jsonl
{"messages": [
    {"role": "system", "content": "你是一个有帮助的AI助手。"},
    {"role": "user", "content": "你好"},
    {"role": "assistant", "content": "你好！有什么我可以帮助你的吗？"}
]}
```

```jsonl
{"messages": [
    {"role": "user", "content": "什么是机器学习？"},
    {"role": "assistant", "content": "机器学习是人工智能的一个分支..."}
]}
```

#### 2. 问答格式 (completions)

适合问答对数据：

```jsonl
{"prompt": "中国的首都是哪里？", "completion": "中国的首都是北京。"}
{"prompt": "Python是什么？", "completion": "Python是一种高级编程语言..."}
```

#### 3. 工具调用格式 (tools)

适合函数调用训练：

```jsonl
{
    "messages": [
        {"role": "user", "content": "北京今天天气怎么样？"},
        {
            "role": "assistant",
            "tool_calls": [
                {
                    "id": "call_123",
                    "type": "function",
                    "function": {
                        "name": "get_weather",
                        "arguments": "{\"location\": \"北京\"}"
                    }
                }
            ]
        }
    ],
    "tools": [
        {
            "type": "function",
            "function": {
                "name": "get_weather",
                "description": "获取指定城市的天气",
                "parameters": {
                    "type": "object",
                    "properties": {
                        "location": {"type": "string"}
                    },
                    "required": ["location"]
                }
            }
        }
    ]
}
```

#### 4. 纯文本格式 (text)

适合继续预训练：

```jsonl
{"text": "这是一段用于训练的文本内容..."}
```

### 使用 Hugging Face 数据集

```bash
# 直接使用已格式化的数据集
mlx_lm.lora \
    --data mlx-community/wikisql \
    --train \
    --model MODEL

# 使用 YAML 配置自定义映射
```

### YAML 配置文件

创建 `config.yaml`：

```yaml
# 数据集配置
data: "./data"

# 模型配置
model: "mlx-community/Qwen2.5-7B-Instruct-4bit"

# 训练配置
iters: 600
batch_size: 4
learning_rate: 1e-5
max_seq_length: 2048

# LoRA 配置
lora_layers: 16
lora_parameters:
  - ["model.layers.", ".mlp.gate_proj"]

# 输出配置
adapter_path: "./adapters"

# 验证配置
val_batches: 50
eval_batches: 10
```

## LoRA 微调

### 基本微调

```bash
mlx_lm.lora \
    --model mlx-community/Qwen2.5-7B-Instruct-4bit \
    --train \
    --data ./data \
    --iters 600 \
    --batch-size 4 \
    --learning-rate 1e-5
```

### 使用配置文件

```bash
mlx_lm.lora --config config.yaml
```

### 常用参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--model` | 基础模型路径 | - |
| `--data` | 数据目录 | - |
| `--train` | 启用训练 | - |
| `--iters` | 训练迭代次数 | 100 |
| `--batch-size` | 批量大小 | 4 |
| `--learning-rate` | 学习率 | 1e-5 |
| `--lora-layers` | LoRA 层数 | 16 |
| `--adapter-path` | 适配器保存路径 | adapters/ |
| `--max-seq-length` | 最大序列长度 | 2048 |

### QLoRA 微调（量化模型）

使用量化模型进行微调，内存占用更低：

```bash
# MLX-LM 会自动检测量化模型并使用 QLoRA
mlx_lm.lora \
    --model mlx-community/Qwen2.5-7B-Instruct-4bit \
    --train \
    --data ./data \
    --iters 600
```

### 全模型微调

```bash
mlx_lm.lora \
    --model mlx-community/Qwen2.5-7B-Instruct-4bit \
    --train \
    --data ./data \
    --fine-tune-type full \
    --iters 600
```

支持的微调类型：`lora`（默认）、`dora`、`full`

```bash
# 计算测试集困惑度

```

## 使用微调后的模型

### 方法1：加载适配器

```bash
mlx_lm.generate \
    --model MODEL \
    --adapter-path adapters \
    --prompt "你的提示词"
```

### 方法2：融合模型

```bash
# 融合适配器到基础模型
mlx_lm.fuse \
    --model MODEL \
    --adapter-path adapters

# 使用融合后的模型
mlx_lm.generate \
    --model ./fused_model \
    --prompt "你的提示词"
```

### Python API 使用

```python
from mlx_lm import load, generate

# 加载模型和适配器
model, tokenizer = load(
    "MODEL_PATH",
    adapter_path="adapters"
)

# 生成文本
response = generate(model, tokenizer, prompt="你好")
print(response)
```

## 高级技巧

### 提示词掩码

只对 completion 计算损失，忽略 prompt：

```bash
mlx_lm.lora \
    --model MODEL \
    --train \
    --data ./data \
    --mask-prompt
```

### 梯度累积

在小内存设备上模拟大批量：

```bash
mlx_lm.lora \
    --model MODEL \
    --train \
    --data ./data \
    --batch-size 1 \
    --grad-accumulation-steps 8
```

### 梯度检查点

用计算换内存：

```bash
mlx_lm.lora \
    --model MODEL \
    --train \
    --data ./data \
    --grad-checkpoint
```

### 恢复训练

```bash
mlx_lm.lora \
    --model MODEL \
    --train \
    --data ./data \
    --resume-adapter-file adapters/adapters.safetensors
```

## 内存优化

如果你的 Mac 内存不足，按以下顺序尝试：

1. **使用量化模型（QLoRA）**
   ```bash
   mlx_lm.lora --model mlx-community/*-4bit --train --data ./data
   ```

2. **减小批量大小**
   ```bash
   --batch-size 1
   ```

3. **减少 LoRA 层数**
   ```bash
   --lora-layers 4
   ```

4. **使用梯度检查点**
   ```bash
   --grad-checkpoint
   ```

5. **梯度累积**
   ```bash
   --batch-size 1 --grad-accumulation-steps 4
   ```

## 完整示例

### 中文问答微调示例

1. 创建 `train.jsonl`：

```jsonl
{"messages": [{"role": "user", "content": "什么是人工智能？"}, {"role": "assistant", "content": "人工智能（AI）是计算机科学的一个分支..."}]}
{"messages": [{"role": "user", "content": "机器学习有哪些类型？"}, {"role": "assistant", "content": "机器学习主要分为监督学习、无监督学习和强化学习..."}]}
```

2. 开始微调：

```bash
mlx_lm.lora \
    --model mlx-community/Qwen2.5-7B-Instruct-4bit \
    --train \
    --data ./data \
    --iters 500 \
    --batch-size 4 \
    --learning-rate 1e-5 \
    --lora-layers 16
```

3. 测试微调结果：

```bash
mlx_lm.generate \
    --model mlx-community/Qwen2.5-7B-Instruct-4bit \
    --adapter-path ./adapters \
    --prompt "什么是深度学习？"
```

## 常见问题

**Q: 微调需要多长时间？**

A: 在 M1 Max 32GB 上，600 次迭代约需 30-60 分钟，具体取决于模型大小和批量大小。

**Q: 我应该选择多少迭代次数？**

A: 从 500-1000 次开始，观察验证损失，如果还在下降可以继续训练。

**Q: 如何判断微调是否成功？**

A: 对比微调前后的输出质量，或者计算验证集困惑度是否下降。
