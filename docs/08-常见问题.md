# 常见问题解答

这里收集了中国 Mac 开发者使用 MLX-LM 时常见的各种问题。

## 安装问题

### Q: pip install mlx-lm 失败，提示找不到包？

**A:** 确认以下几点：

1. 你使用的是 Apple 芯片 Mac（M1/M2/M3/M4）
2. macOS 版本至少是 12.0
3. pip 版本是最新的

```bash
# 更新 pip
pip install --upgrade pip

# 检查系统
uname -m  # 应显示 arm64
sw_vers   # 查看 macOS 版本
```

### Q: 安装时提示权限错误？

**A:** 使用虚拟环境或 `--user` 参数：

```bash
# 推荐：使用虚拟环境
python3 -m venv mlx-env
source mlx-env/bin/activate
pip install mlx-lm

# 或者使用 --user
pip install --user mlx-lm
```

### Q: MLX 版本不兼容？

**A:** 更新 MLX：

```bash
pip install --upgrade mlx mlx-lm
```

## 运行问题

### Q: 模型生成速度很慢？

**A:** 可能的原因和解决方案：

1. **模型太大**：使用量化模型
   ```bash
   # 使用 4-bit 量化版本
   mlx_lm.generate --model mlx-community/Qwen2-7B-Instruct-4bit
   ```

2. **内存不足**：关闭其他应用
   ```bash
   # 查看内存使用
   htop
   ```

3. **系统限制**：增加有线内存（macOS 15+）
   ```bash
   sudo sysctl iogpu.wired_limit_mb=24000
   ```

### Q: 提示 "WARNING: Generating with a model that requires..."？

**A:** 这表示模型较大，可能运行较慢。尝试：

1. 使用量化版本
2. 减少 `--max-kv-size`
3. 增加系统有线内存限制

### Q: 运行时出现 "Killed" 或崩溃？

**A:** 通常是内存不足：

1. 使用更小的模型
2. 使用量化模型
3. 减小批量大小
4. 关闭其他应用

## 模型问题

### Q: 如何选择合适的中文模型？

**A:** 根据你的需求：

| 需求 | 推荐模型 |
|------|----------|
| 日常对话 | Qwen2.5-7B-Instruct-4bit |
| 复杂任务 | Qwen2.5-14B-Instruct-4bit |
| 代码生成 | Qwen2.5-Coder-7B-Instruct-4bit |
| 英中双语 | Gemma-2-9b-it-4bit |

### Q: 模型下载失败怎么办？

**A:** 尝试以下方法：

1. 使用镜像站点
   ```bash
   export HF_ENDPOINT=https://hf-mirror.com
   ```

2. 手动下载后转换
   ```bash
   # 从其他渠道下载模型文件
   # 然后使用本地路径转换
   mlx_lm.convert --model /path/to/local/model
   ```

3. 使用 MLX Community 预转换的模型
   ```bash
   mlx_lm.generate --model mlx-community/Qwen2-7B-Instruct-4bit
   ```

### Q: 提示 "trust_remote_code" 警告？

**A:** 某些模型需要信任远程代码：

```bash
# 命令行
mlx_lm.generate --model qwen/Qwen-7B --trust-remote-code

# Python API
model, tokenizer = load(
    "qwen/Qwen-7B",
    tokenizer_config={"trust_remote_code": True}
)
```

## 中文支持

### Q: 生成的中文质量不好？

**A:** 尝试：

1. 使用专门的中文模型
   ```bash
   --model Qwen/Qwen2-7B-Instruct
   ```

2. 在提示词中明确要求中文
   ```python
   prompt = "请用中文回答：什么是人工智能？"
   ```

3. 设置合适的温度参数
   ```bash
   --temp 0.7
   ```

### Q: 支持哪些中文模型？

**A:** MLX-LM 支持的中文模型包括：

- **Qwen 系列**（推荐）：Qwen、Qwen2、Qwen2.5
- **Qwen Coder**：代码生成
- **Baichuan**：百川系列
- **ChatGLM**：清华 ChatGLM

## 性能优化

### Q: 16GB 内存能运行什么模型？

**A:** 推荐配置：

- **7B 模型 4-bit 量化**：可以运行
- **3B-4B 模型**：可以运行 FP16
- **14B+ 模型**：不建议

```bash
# 16GB 内存推荐
mlx_lm.generate --model mlx-community/Qwen2-7B-Instruct-4bit
```

### Q: 如何提高推理速度？

**A:** 优化建议：

1. 使用量化模型（4-bit）
2. 减小 `--max-kv-size`
3. 使用较小的模型
4. 升级到 macOS 15+

### Q: 可以同时运行多个模型吗？

**A:** 可以，但需要注意内存：

```python
# 可以加载多个模型
model1, tokenizer1 = load("model1")
model2, tokenizer2 = load("model2")

# 但会占用大量内存，谨慎使用
```

## 微调问题

### Q: 微调时内存不足？

**A:** 按顺序尝试：

1. 使用 QLoRA（量化模型）
2. 减小批量大小：`--batch-size 1`
3. 减少微调层数：`--lora-layers 4`
4. 使用梯度检查点：`--grad-checkpoint`
5. 梯度累积：`--grad-accumulation-steps 4`

### Q: 微调需要多长时间？

**A:** 参考时间（M1 Max 32GB）：

- 7B 模型，600 次迭代：约 30-60 分钟
- 14B 模型，600 次迭代：约 1-2 小时

### Q: 如何判断微调效果？

**A:** 评估方法：

1. 观察验证损失下降
2. 手动测试输出质量
3. 计算测试集困惑度
4. 对比微调前后的表现

## 其他问题

### Q: MLX-LM 与 Ollama 对比？

**A:** 各有优势：

| 特性 | MLX-LM | Ollama |
|------|---------|--------|
| 性能 | 原生优化 | 良好 |
| 内存效率 | 很高 | 高 |
| 微调支持 | ✅ | ❌ |
| 模型数量 | 多 | 很多 |
| 易用性 | 需要一些代码 | 非常简单 |

### Q: 可以在服务器上运行吗？

**A:** MLX 主要为 Mac 设计，但：
- 有 Linux/CUDA 版本（实验性）
- 推荐 Mac 开发，部署到其他平台

### Q: 如何获取帮助？

**A:** 获取帮助的渠道：

1. [GitHub Issues](https://github.com/ml-explore/mlx-lm/issues)
2. [MLX Discord](https://discord.gg/mLx)
3. Hugging Face 论坛

### Q: 有图形界面吗？

**A:** 目前主要是命令行和 Python API，但可以：
- 使用 `mlx_lm.server` 启动 Web 服务
- 自行构建 Web 界面
- 使用第三方 GUI 工具

## 错误代码速查

| 错误 | 原因 | 解决方案 |
|------|------|----------|
| `Killed` | 内存不足 | 使用更小的量化模型 |
| `No matching distribution` | 非 Apple 芯片 | 使用 Apple Silicon Mac |
| `CUDA error` | 尝试使用 CUDA | MLX 不需要 CUDA |
| `File not found` | 模型路径错误 | 检查模型路径 |
| `Permission denied` | 权限问题 | 使用虚拟环境 |
