# 长文本处理技巧

处理长文本时，MLX-LM 提供了多种优化方法来提高效率和降低内存使用。

## KV 缓存

KV 缓存（Key-Value Cache）用于存储中间计算结果，避免重复计算。

### 使用旋转 KV 缓存

```bash
mlx_lm.generate \
    --prompt "长文本..." \
    --max-kv-size 2048
```

### KV 大小建议

| max-kv-size | 内存使用 | 质量 |
|-------------|----------|------|
| 512 | 很低 | 较差 |
| 1024 | 低 | 一般 |
| 2048 | 中等 | 良好 |
| 4096 | 高 | 很好 |
| 8192+ | 很高 | 优秀 |

```bash
# 低内存配置（适合 16GB）
mlx_lm.generate --prompt "..." --max-kv-size 1024

# 高质量配置（适合 32GB+）
mlx_lm.generate --prompt "..." --max-kv-size 4096
```

## 提示词缓存

当你需要多次使用相同的长文本作为上下文时，提示词缓存可以大幅提升速度。

### 缓存提示词

```bash
# 从文件缓存
cat long_context.txt | mlx_lm.cache_prompt \
    --model Qwen/Qwen2.5-7B-Instruct \
    --prompt - \
    --prompt-cache-file context_cache.safetensors

# 直接指定
mlx_lm.cache_prompt \
    --model Qwen/Qwen2.5-7B-Instruct \
    --prompt "这是一段很长的上下文..." \
    --prompt-cache-file context_cache.safetensors
```

### 使用缓存的提示词

```bash
mlx_lm.generate \
    --prompt-cache-file context_cache.safetensors \
    --prompt "\n请总结以上内容。"
```

缓存会作为前缀自动添加到你的提示词之前。

### Python API 使用缓存

在 Python 中使用提示词缓存需要手动处理：

```python
from mlx_lm import load, generate
from mlx_lm.models.cache import make_prompt_cache, save_prompt_cache, load_prompt_cache
import mlx.core as mx

# 准备长文本

prompt = tokenizer.apply_chat_template(

# 创建缓存
prompt_cache = make_prompt_cache(model, max_kv_size=2048)

# 处理提示词（填充缓存）
y = mx.array(tokenizer.encode(prompt))
for _ in range(len(y)):
    # 这里需要使用 generate_step 来填充缓存
    # 简化起见，建议使用命令行工具 mlx_lm.cache_prompt
    pass

# 保存缓存
save_prompt_cache("context_cache.safetensors", prompt_cache)
```

**更简单的方式**：使用命令行工具创建缓存，然后在 Python 中加载使用：

```bash
# 先用命令行创建缓存
echo "长文本内容..." | mlx_lm.cache_prompt \
    --model Qwen/Qwen2.5-7B-Instruct \
    --prompt - \
    --prompt-cache-file context_cache.safetensors
```

```python
from mlx_lm import load, generate, stream_generate
from mlx_lm.models.cache import load_prompt_cache

# 加载模型和缓存
model, tokenizer = load("mlx-community/Qwen2.5-7B-Instruct-4bit")
prompt_cache = load_prompt_cache("context_cache.safetensors")

# 使用缓存生成

for response in stream_generate(
    model, tokenizer, prompt,

    max_tokens=200
```

## 分块处理

对于超长文本，可以将其分块处理。

### 文档摘要示例

```python
from mlx_lm import load, generate

model, tokenizer = load("mlx-community/Qwen2.5-7B-Instruct-4bit")

def chunk_text(text: str, chunk_size: int = 3000) -> list:
    """将文本分块"""
    words = text.split()
    chunks = []
    for i in range(0, len(words), chunk_size):
        chunk = " ".join(words[i:i + chunk_size])
        chunks.append(chunk)
    return chunks

def summarize_long_document(text: str) -> str:
    """分块摘要长文档"""
    chunks = chunk_text(text)

    # 先摘要每个块
    chunk_summaries = []
    for i, chunk in enumerate(chunks):
        prompt = f"请用一句话总结以下内容：\n\n{chunk}"
        summary = generate(model, tokenizer, prompt=prompt, max_tokens=100)
        chunk_summaries.append(f"第{i+1}部分：{summary}")

    # 再整合所有摘要
    combined = "\n".join(chunk_summaries)
    final_prompt = f"请整合以下摘要，形成完整的文档摘要：\n\n{combined}"
    final_summary = generate(model, tokenizer, prompt=final_prompt, max_tokens=500)

    return final_summary

# 使用
long_doc = "这里是长文档..."
summary = summarize_long_document(long_doc)
print(summary)
```

## 滑动窗口

对于需要保持上下文连续性的场景，使用滑动窗口：

```python
from mlx_lm import load, generate

model, tokenizer = load("mlx-community/Qwen2.5-7B-Instruct-4bit")

def sliding_window_process(text: str, window_size: int = 2000, overlap: int = 500):
    """使用滑动窗口处理长文本"""
    words = text.split()
    results = []

    for i in range(0, len(words), window_size - overlap):
        window = " ".join(words[i:i + window_size])

        prompt = f"请处理以下文本（保持上下文连贯性）：\n\n{window}"
        result = generate(model, tokenizer, prompt=prompt, max_tokens=300)
        results.append(result)

    return results
```

## RAG（检索增强生成）

对于大量文档，使用 RAG 可以只检索相关部分：

```python
from mlx_lm import load, generate
import numpy as np

model, tokenizer = load("mlx-community/Qwen2.5-7B-Instruct-4bit")

# 简化的 RAG 示例
class SimpleRAG:
    def __init__(self, model_path: str):
        self.model, self.tokenizer = load(model_path)
        self.documents = []

    def add_document(self, doc: str):
        self.documents.append(doc)

    def retrieve(self, query: str, top_k: int = 3):
        # 简化版：直接返回前几个文档
        # 实际应用应使用嵌入向量相似度搜索
        return self.documents[:top_k]

    def answer(self, query: str):
        # 检索相关文档
        relevant_docs = self.retrieve(query)

        # 构建提示词
        context = "\n\n".join(relevant_docs)
        prompt = f"根据以下文档回答问题：\n\n{context}\n\n问题：{query}"

        # 生成答案
        response = generate(
            self.model,
            self.tokenizer,
            prompt=prompt,
            max_tokens=500
        )

        return response
```

### 场景2：代码库分析

```python
from mlx_lm import load, stream_generate
from mlx_lm.models.cache import make_prompt_cache
from mlx_lm.sample_utils import make_sampler

model, tokenizer = load("mlx-community/Qwen2.5-Coder-7B-Instruct-4bit")
sampler = make_sampler(temp=0.3)

# 创建缓存
prompt_cache = make_prompt_cache(model)

# 准备代码库上下文
code_context = """
这是一个Python项目，包含以下模块：
- user.py: 用户管理
- auth.py: 认证逻辑
- database.py: 数据库操作
"""

# 将上下文添加到缓存
messages = [{"role": "system", "content": code_context}]
prompt = tokenizer.apply_chat_template(messages, add_generation_prompt=False)
# 注意：需要先运行 generate 来填充缓存，这里简化处理

# 基于上下文提问
questions = [
    "如何添加新用户？",
    "认证流程是怎样的？",
]

for q in questions:
    messages_q = [
        {"role": "system", "content": code_context},
        {"role": "user", "content": q}
    ]
    prompt_q = tokenizer.apply_chat_template(messages_q, add_generation_prompt=True)

    for response in stream_generate(
        model, tokenizer, prompt_q,
        sampler=sampler,
        prompt_cache=prompt_cache,
        max_tokens=300
    ):
        print(response.text, end="", flush=True)
    print("\n")
```

### 场景3：多文档总结

```python
from mlx_lm import load, generate

model, tokenizer = load("mlx-community/Qwen2.5-7B-Instruct-4bit")

def multi_doc_summary(docs: list) -> str:
    # 1. 总结每个文档
    summaries = []
    for i, doc in enumerate(docs):
        prompt = f"请总结第{i+1}个文档：\n\n{doc[:2000]}"
        summary = generate(model, tokenizer, prompt=prompt, max_tokens=200)
        summaries.append(summary)

    # 2. 综合总结
    all_summaries = "\n".join([f"文档{i+1}摘要：{s}" for i, s in enumerate(summaries)])
    final_prompt = f"请综合以下摘要，生成整体总结：\n\n{all_summaries}"
    final = generate(model, tokenizer, prompt=final_prompt, max_tokens=500)

    return final
```

## 常见问题

**Q: KV 缓存会影响输出质量吗？**

A: 会。较小的 KV 大小可能导致模型"忘记"较早的内容。选择合适的 KV 大小是内存和质量的权衡。

**Q: 提示词缓存占用多少空间？**

A: 通常为几十到几百 MB，取决于缓存的文本长度。

**Q: 可以共享缓存文件吗？**

A: 可以，但缓存文件与特定模型绑定。不同模型不能共享缓存。
