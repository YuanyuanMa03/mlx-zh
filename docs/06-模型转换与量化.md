# 模型转换与量化

本指南介绍如何将 Hugging Face 模型转换为 MLX 格式，以及如何进行模型量化。

## 为什么需要转换？

MLX-LM 使用特殊的模型格式以优化 Apple 芯片的性能。直接使用 Hugging Face 模型需要先转换。

## 基本转换

### 转换命令

```bash
mlx_lm.convert --model Qwen/Qwen2.5-7B-Instruct
```

这将：
1. 从 Hugging Face 下载模型
2. 转换为 MLX 格式
3. 保存到 `./mlx_model` 目录

### 指定输出路径

```bash
mlx_lm.convert \
    --model Qwen/Qwen2.5-7B-Instruct \
    --mlx-path ./my_models/qwen2-7b
```

## 模型量化

量化可以大幅减少模型内存占用，同时保持较好的质量。

### 4-bit 量化（推荐）

```bash
mlx_lm.convert --model Qwen/Qwen2.5-7B-Instruct -q
```

### 量化效果对比

| 模型 | FP16 大小 | 4-bit 大小 | 内存节省 |
|------|-----------|------------|----------|
| Qwen2.5-7B | ~14GB | ~4GB | 72% |
| Qwen2-14B | ~28GB | ~8GB | 72% |
| Qwen2-32B | ~64GB | ~18GB | 72% |

### 指定量化参数

```bash
mlx_lm.convert \
    --model Qwen/Qwen2.5-7B-Instruct \
    -q \
    --q-bits 4
```

## 上传到 Hugging Face

### 基本上传

```bash
# 1. 登录 Hugging Face
huggingface-cli login

# 2. 转换并上传
mlx_lm.convert \
    --model Qwen/Qwen2.5-7B-Instruct \
    -q \
    --upload-repo your-username/qwen2-7b-mlx-4bit
```

### 完整示例

```bash
mlx_lm.convert \
    --model Qwen/Qwen2.5-7B-Instruct \
    --mlx-path ./mlx_model \
    -q \
    --upload-repo your-username/my-qwen-model \
    --hf-path Qwen/Qwen2.5-7B-Instruct \
    --commit-message "Upload Qwen2 7B 4-bit MLX model"
```

参数说明：
- `--upload-repo`: 上传目标仓库
- `--hf-path`: 原始模型仓库（用于署名）
- `--commit-message`: 提交信息

## 高级量化方法

### AWQ 量化

```bash
pip install "mlx-lm[train]"

mlx_lm.awq --model ./mlx_model
```

### GPTQ 量化

```bash
pip install "mlx-lm[train]"

mlx_lm.gptq --model ./mlx_model
```

### 动态量化

```bash
mlx_lm.dynamic_quant --model ./mlx_model
```

## 从本地模型转换

如果你有本地下载的模型：

```bash
mlx_lm.convert --model /path/to/local/model
```

## 转换融合后的模型

```bash
# 融合并转换为 GGUF
mlx_lm.fuse \
    --model MODEL \
    --adapter-path adapters \
    --export-gguf \
    --gguf-path ./output/model.gguf
```

> 注意：GGUF 导出目前仅支持 Mistral、Mixtral 和 Llama 风格模型的 FP16 精度。

## MLX Community 模型

许多热门模型已经被 MLX 社区转换好：

```bash
# 直接使用转换好的模型
mlx_lm.generate --model mlx-community/Qwen2.5-7B-Instruct-4bit
```

热门 MLX Community 模型：
- `mlx-community/Qwen2.5-7B-Instruct-4bit`
- `mlx-community/Qwen2.5-7B-Instruct-4bit`
- `mlx-community/Llama-3.2-3B-Instruct-4bit`
- `mlx-community/gemma-2-9b-it-4bit`

## 常见问题

### Q: 转换需要多长时间？

A: 取决于模型大小和网络速度。7B 模型约需 10-30 分钟（包括下载）。

### Q: 转换失败怎么办？

A: 检查以下几点：
1. 确认有足够的磁盘空间
2. 确认网络连接正常
3. 某些模型需要 `--trust-remote-code` 参数

### Q: 如何验证转换后的模型？

A:
```bash
# 测试加载
python -c "from mlx_lm import load; load('./mlx_model')"

# 测试生成
mlx_lm.generate --model ./mlx_model --prompt "你好"
```

### Q: 可以转换任意 Hugging Face 模型吗？

A: MLX-LM 支持许多流行模型架构，但不是全部。查看[支持的模型列表](https://github.com/ml-explore/mlx-lm#supported-models)。

## 完整工作流示例

### 1. 转换新模型

```bash
# 下载并转换 Qwen2.5-7B
mlx_lm.convert \
    --model Qwen/Qwen2.5-7B-Instruct \
    -q \
    --mlx-path ./models/qwen2.5-7b-4bit
```

### 2. 测试模型

```bash
mlx_lm.generate \
    --model ./models/qwen2.5-7b-4bit \
    --prompt "你好" \
    --max-tokens 100
```

### 3. 上传到 Hugging Face

```bash
mlx_lm.convert \
    --model Qwen/Qwen2.5-7B-Instruct \
    -q \
    --upload-repo your-username/qwen2.5-7b-mlx-4bit
```

### 4. 使用上传的模型

```bash
mlx_lm.generate \
    --model your-username/qwen2.5-7b-mlx-4bit \
    --prompt "你好"
```

## 存储空间要求

| 操作 | 7B 模型 | 14B 模型 | 32B 模型 |
|------|---------|----------|----------|
| 原始下载 | ~14GB | ~28GB | ~64GB |
| 转换后 FP16 | ~14GB | ~28GB | ~64GB |
| 转换后 4-bit | ~4GB | ~8GB | ~18GB |

建议预留模型大小 2 倍的磁盘空间用于转换过程。
